# Changelog

## [v0.1.0] - 2026-04-16

### 新功能
- **喝水月曆**：右上角月曆按鈕，可查看每月每天的喝水量與達標狀況（綠色達標、橘色未達標、灰色無紀錄），支援月份切換
- **容器快速編輯**：每個容器卡片右上角直接顯示鉛筆按鈕，點擊即可修改圖示、名稱、容量，或刪除容器

### 問題修復

#### PWA 安裝版無法收到更新
- **問題**：把 APP 加入主畫面後，即使網站已更新，APP 版依然顯示舊版內容
- **原因**：舊的 Service Worker 還在控制 APP，新的 SW 雖然安裝完成但頁面沒有重載
- **解法**：加入 `controllerchange` 事件監聽，新 SW 接管時自動重載一次頁面

#### 每次更新都需要手動升版 SW cache 版本號
- **問題**：每改一次程式就要手動把 `sw.js` 裡的 `'drinky-v1'` 改成 `v2`、`v3`...
- **原因**：原本採用 cache-first 策略，SW 永遠優先回傳快取，沒改版本號就拿不到新檔案
- **解法**：改為 network-first 策略。有網路時永遠從伺服器抓最新版，快取只作離線備用，自動覆蓋舊版，版本號不再需要

#### 首次開啟 APP 兩個彈窗同時出現且無法關閉
- **問題**：第一次打開時「手動輸入」和「容器管理」彈窗同時跳出，按叉叉無效，只有 Ctrl+Shift+R 才能清掉
- **原因**：SW 快取了早期版本的 HTML（彈窗沒有 `hidden` 屬性），每次開啟都讀到這份舊快取
- **解法**：改為 network-first（根本解）+ JS 初始化時強制關閉所有彈窗（防禦性設計）
